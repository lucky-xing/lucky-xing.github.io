一、引言：三篇论文的共同使命

2026年，三篇关于Transformer架构改进的重要论文相继发布，它们从不同角度解决了大语言模型面临的核心挑战：

📚 三篇论文概览

  • Attention Residuals Kimi Team - 用注意力机制替代固定残差权重
  • MoDA 字节跳动 - 让注意力访问历史层的KV缓存
  • ComprExIT Jiangnan Ye等 - 显式信息传输的上下文压缩

这三篇论文虽然具体方法不同，但都指向一个共同的根本问题：Transformer中的信息流动效率问题。本文将深入对比分析它们的异同。

二、问题背景对比

论文 核心问题 问题表现
Attention Residuals PreNorm的均匀聚合问题 浅层特征被稀释，隐藏状态幅度随深度增长
MoDA 深度增加时的信号退化 浅层有效特征在深层难以恢复
ComprExIT 长上下文压缩的信息丢失 表示覆盖 + 缺乏全局协调

2.1 Attention Residuals：PreNorm的稀释效应

在PreNorm架构中，每层的输出以固定权重（=1）累加到最终结果：

h_L = h_0 + Σ_{i=1}^{L} Norm(F_i(h_{i-1}))

这导致：

  • 浅层特征被后期层的输出"淹没"
  • 隐藏状态幅度随深度O(√L)增长
  • 梯度分布不均，深层学习快、浅层学习慢

2.2 MoDA：深度缩放的信号退化

随着LLM变深，在浅层形成的有效特征经过多次残差更新后逐渐被稀释。这与Attention Residuals问题同根同源，但关注点略有不同：

  • MoDA更关注"历史层的信息"能否被有效利用
  • 传统方法中，每层只能看到当前层的输入
  • MoDA让每层可以"回溯"历史层的KV缓存

2.3 ComprExIT：上下文压缩的两个缺陷

ComprExIT发现现有压缩方法的两个根本问题：
  • 表示覆盖问题：压缩token在早期层聚合的信息可能被后续层更新覆盖
  • 缺乏全局协调：各压缩token独立进行注意力聚合，没有协调机制导致信息冗余和遗漏

三、核心方法对比

论文 核心创新 聚合方式
Attention Residuals 伪查询注意力机制 输入依赖的选择性聚合
MoDA 深度KV缓存 + 序列KV缓存 Combined Softmax
ComprExIT 深度传输 + 宽度传输 最优传输全局优化

3.1 Attention Residuals：伪查询机制

用可学习的伪查询向量替代固定残差权重：

核心公式

h_l = Σ_{i=0}^{l-1} α_{i→l} · v_i

其中 α_{i→l} = softmax(v_i · w_l / √d)

每个层l有一个可学习的伪查询 w_l，来决定关注哪些前续层

Block AttnRes（分块优化）

为解决O(L²d)计算开销，将层划分为N个块：

  • 块内：标准PreNorm（保持效率）
  • 块间：注意力聚合（选择性）

3.2 MoDA：深度KV缓存

让注意力头同时访问两类KV：

Combined Softmax

Attention(Q, K_seq, V_seq, K_depth, V_depth) = softmax(Q·[K_seq, K_depth]^T / √d) · [V_seq, V_depth]

  • 序列KV：当前层的输入token
  • 深度KV：存储前几层的KV缓存
  • 每个头可动态决定关注深度还是序列

3.3 ComprExIT：显式信息传输

沿两个正交维度传输信息：

（1）深度传输

沿层维度选择性传输：

h̃_t = Σ_{ℓ} α_{t,ℓ} · W_a · h_t^{(ℓ)}

（2）宽度传输

通过最优传输全局协调：

min Σ_{t,k} Π_{t,k} · C_{t,k}

使用Sinkhorn算法高效求解

四、适用场景对比

论文 最佳适用场景 模型规模 与现有架构兼容性
Attention Residuals 深层模型、多步推理任务 1B-48B+ 需修改残差连接
MoDA 超长上下文、长序列建模 700M-1.5B 需修改注意力机制
ComprExIT 长上下文问答、信息检索 1B-3B 冻结LLM + 轻量模块

4.1 Attention Residuals

最佳场景：需要深层推理的任务，尤其是：
  • 研究生水平问答（GPQA-Diamond +20.3%）
  • 数学推理（+6.7%）
  • 代码生成（+5.2%）

这些任务需要模型"记住"并有效利用早期推理步骤的信息，AttnRes的选择性聚合完美契合这一需求。

4.2 MoDA

最佳场景：需要利用历史层信息的任务：
  • 长序列语言建模
  • 需要"回顾"早期上下文的生成任务
  • 与post-norm结合效果更佳

4.3 ComprExIT

最佳场景：长上下文压缩场景：
  • 4倍压缩率下仍保持高性能
  • 6个QA基准一致超越基线
  • 小模型（1B）上性能稳定

六、核心思想对比

🔍 深度剖析：三篇论文的"选择"哲学

  • AttnRes = 选择性聚合 - 不再均匀对待每一层，让网络自己决定关注哪些层
  • MoDA = 历史记忆 - 不再只关注当前层，让网络可以"查阅"历史层的记忆
  • ComprExIT = 全局优化 - 不再让各部分独立决策，通过全局优化协调信息分配

6.1 信息的"流动性"对比

论文 信息聚合维度 聚合方式 可解释性
AttnRes 层 → 层 注意力加权 伪查询可视化
MoDA 序列 + 深度 Combined Softmax 注意力热力图
ComprExIT Token + 层 最优传输 传输计划可视化

6.2 工程实现的权衡

论文 计算复杂度 内存开销 工程难度
AttnRes O(L²d) → O(Ld×N_block) O(Ld) 中
MoDA ~1.037x FLOPs 深度KV缓存 高（Triton优化）
ComprExIT 压缩后减少4x Sinkhorn迭代 低（冻结LLM）

七、总结与选择指南

7.1 如何选择合适的方法？

选择依据：
  • 如果你的模型层数很深（40+层），且需要多步推理 → 选择 Attention Residuals
  • 如果你的场景是长序列建模，需要利用历史信息 → 选择 MoDA
  • 如果你需要压缩长上下文进行问答/检索 → 选择 ComprExIT

7.2 三篇论文的统一视角

从更高视角看，这三篇论文都在解决Transformer中的"信息流动效率"问题：

  • 传统Transformer：信息逐层传递，但无法选择性保留
  • AttnRes：让每层可以"投票"决定信息权重
  • MoDA：让每层可以"查阅"历史"笔记"
  • ComprExIT：让压缩器可以"全局规划"信息分配

7.3 未来展望

这些工作揭示了Transformer架构的巨大优化空间。未来可能的融合方向包括：

  • 将MoDA的历史KV访问机制与AttnRes的选择性聚合结合
  • 将ComprExIT的最优传输思想引入层间信息分配
  • 在更大规模模型（7B+）上验证这些方法的可扩展性
