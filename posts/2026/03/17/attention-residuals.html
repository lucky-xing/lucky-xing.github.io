Attention Residuals: 重新定义Transformer的残差连接

引言：从ResNet到PreNorm的演进

残差连接（Residual Connections）自从在ResNet中被提出以来，就成为了深度学习架构中最核心的设计之一。在Transformer出现后，残差连接同样被广泛应用，成为了构建深层Transformer不可或缺的基础组件。

现代大语言模型（LLM）普遍采用两种残差连接变体：PostNorm 和 PreNorm。PostNorm将层归一化放在残差分支之后，而PreNorm则将层归一化放在残差分支之前。DeepMind的论文"Understanding the Difficulty of Training Deep Transformers"表明，PreNorm相比PostNorm更容易训练，但这种优势并非没有代价。

PreNorm的隐藏问题：稀释效应

尽管PreNorm在训练稳定性方面表现更好，但它存在一个被广泛忽视的根本性问题：随着模型深度增加，所有层的输出以固定权重（=1）进行累加，导致：

  • 隐藏状态幅度无限增长：每一层的输出都会直接加到累积和上，层数越多，最终的隐藏状态向量范数越大
  • 浅层特征被稀释：后期层看到的"信号"是前面所有层输出的混合，早期层学到的有意义的表示被后来的噪声稀释
  • 各层贡献不均衡：深层对最终表示的贡献远大于浅层，但这种贡献是"均匀"而非"选择性的"

PreNorm的数学表达式

在标准PreNorm Transformer中，第l层的输出可以表示为：

h_l = h_{l-1} + Norm(F_{l}(h_{l-1}))

经过L层后，最终输出为：

h_L = h_0 + Σ_{i=1}^{L} Norm(F_i(h_{i-1}))

每一项的权重固定为1，这就是所谓的均匀聚合（Uniform Aggregation）。

核心创新：Attention Residuals

来自Kimi Team的最新论文Attention Residuals (AttnRes)提出了一个优雅的解决方案：用注意力机制替代固定的残差累加权重，让每个层能够输入依赖的、内容感知的方式选择性聚合前面的表示。

AttnRes的核心公式

AttnRes将固定权重累加替换为注意力加权：

h_l = Σ_{i=0}^{l-1} α_{i→l} · v_i

其中：

  • v_i = 第i层的输出表示
  • α_{i→l} = 从层i到层l的注意力权重（通过softmax归一化）

关键创新在于：权重α_{i→l}是通过一个可学习的伪查询向量w_l计算得出，实现了真正的输入依赖性。

注意力权重的计算方式

具体而言，对于第l层，AttnRes使用以下方式计算注意力权重：

伪查询注意力机制

为每一层l引入一个可学习的伪查询向量w_l ∈ ℝ^d：

logit_{i→l} = (v_i · w_l) / √d

α_{i→l} = softmax_i(logit_{i→l})

这样，每一层都可以"询问"前面的层："谁对我的输出最有贡献？"，而不是简单地将所有贡献等权相加。

两种实现方案

1. Full AttnRes（完整版）

最直接的实现方式：每一层都可以 attends 到之前所有层的输出。这种方法理论上效果最好，但内存开销为 O(Ld)，其中L是层数，d是隐藏维度。在超大模型上这会成为瓶颈。

2. Block AttnRes（实用版）

为解决内存问题，论文提出了分块版本的AttnRes：

  • 将L层划分为N个块（block）
  • 块内使用标准残差连接（保持局部信息流动）
  • 块之间使用注意力聚合（实现跨块选择性）
  • 只需要 O(Nd) 的额外内存，其中N是块数（通常约8个）

(a) 标准残差：均匀的加法累加 (b) Full AttnRes：每层 attends 到所有前续输出 (c) Block AttnRes：块内标准残差，块间注意力聚合

Block AttnRes的PyTorch伪代码

def block_attn_res(blocks: list[Tensor], partial_block: Tensor, proj: Linear, norm: RMSNorm) -> Tensor:
    """
    块间注意力：attend 到各块表示 + 当前部分和
    blocks: 已完成的N个块表示 [B, T, D]
    partial_block: 块内部分和 [B, T, D]
    """
    V = torch.stack(blocks + [partial_block])  # [N+1, B, T, D]
    K = norm(V)
    logits = torch.einsum('d, n b t d -> n b t', proj.weight.squeeze(), K)
    h = torch.einsum('n b t, n b t d -> b t d', logits.softmax(0), V)
    return h

def forward(self, blocks, hidden_states):
    partial_block = hidden_states
    
    # 在attention前应用block attnres
    h = block_attn_res(blocks, partial_block, self.attn_res_proj, self.attn_res_norm)

    # 到达块边界时，开始新块
    if self.layer_number % (self.block_size // 2) == 0:
        blocks.append(partial_block)
        partial_block = None

    # 自注意力层
    attn_out = self.attn(self.attn_norm(h))
    partial_block = partial_block + attn_out if partial_block is not None else attn_out

    # MLP前再次应用block attnres
    h = block_attn_res(blocks, partial_block, self.mlp_res_proj, self.mlp_res_norm)

    # MLP层
    mlp_out = self.mlp(self.mlp_norm(h))
    partial_block = partial_block + mlp_out

    return blocks, partial_block

实验结果深度解析

1. 扩展定律（Scaling Laws）

论文在1M到200M参数规模的模型上进行了系统性的扩展定律实验，测试了不同计算预算（FLOPs）下的验证集loss。

AttnRes (橙色) vs Baseline (蓝色) 在不同计算预算下的 loss 对比

关键发现：

  • AttnRes在所有计算预算下都一致优于基线
  • Block AttnRes的性能相当于用1.25x更多计算训练的基线模型
  • 这意味着相同的计算资源，使用AttnRes可以获得~25%的性能提升

2. 下游任务性能

论文在Kimi Linear 48B模型（3B activated，训练1.4T tokens）上进行了广泛的下游任务评估：

关键洞察

  • 最大收益任务：GPQA-Diamond（多步推理）提升了7.5个百分点，这是所有任务中最大的收益
  • 代码生成：HumanEval提升3.1，显示出AttnRes对需要精确执行的复杂任务特别有效
  • 数学推理：Math提升3.6，表明模型在多步推理能力上有显著增强
  • 全面提升：所有9个基准都有提升，没有出现负向任务

3. 训练动态分析

论文深入分析了AttnRes对训练过程的影响：

左图：输出幅度随层数变化；右图：梯度范数随层数变化。AttnRes有效控制了幅度增长和梯度分布

输出幅度控制

在标准PreNorm模型中，输出幅度随层数指数增长，这会导致：

  • 数值不稳定（需要更小的学习率）
  • 训练后期难以收敛
  • 推理时可能的数值溢出

AttnRes通过注意力加权自然地实现了幅度归一化：所有权重经过softmax后总和为1，因此最终的输出幅度是有界的。

梯度分布优化

标准PreNorm中，深层的梯度范数远大于浅层，导致：

  • 浅层参数更新不足
  • 深层容易过拟合
  • 训练早期浅层学不到有用的表示

AttnRes使梯度在各层之间分布更均匀，每一层都能得到充分的训练信号。

技术细节与实现考量

1. 伪查询向量的设计

每个层l有一个独立的可学习参数w_l（维度为d），这个设计：

  • 参数量：L × d（L是层数，d是隐藏维度）
  • 相比整体模型参数量可忽略不计
  • 每个层可以学习"关注"不同时期特征的偏好

2. 块大小的选择

论文推荐约8个块（block），这意味着：

  • 块大小 = L / 8（L是总层数）
  • 对于40层模型，块大小约为5层
  • 可以在几乎无额外开销的情况下恢复大部分Full AttnRes的收益

3. 即插即用特性

AttnRes一个重要优势是不需要改变模型架构，只需要：

  • 添加伪查询参数
  • 修改残差连接的聚合方式
  • 保持其他组件不变

这使得它可以轻松集成到现有的预训练流程中。

总结与展望

Attention Residuals提出了一种简单而深刻的改进方案：将残差连接中的固定权重替换为注意力加权。这个改变虽然微小，但带来了显著的性能提升：

  • 理论贡献：首次揭示了PreNorm中"均匀聚合"的问题，并用"选择性聚合"来解决
  • 工程价值：Block AttnRes在几乎无额外开销的情况下实现~25%计算等效收益
  • 实践经验：在真实的大规模模型（48B参数）上验证有效
  • 即插即用：可以轻松集成到现有架构中

这篇论文也提醒我们：在深度学习领域，一些看似"理所当然"的设计（如固定权重残差）可能隐藏着可以优化的空间。AttnRes的成功源于它对这一基础组件的重新思考。
