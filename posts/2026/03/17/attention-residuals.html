一、引言：从ResNet到PreNorm的演进与困境

1.1 残差连接的起源与发展

残差连接（Residual Connections）最初在2015年的ResNet中被提出，旨在解决深层卷积网络训练困难的问题。其核心思想是让网络学习输入与输出之间的"残差"而非直接映射：

ResNet残差模块

y = F(x, {W_i}) + x

其中F是学习到的残差函数，x是输入，y是输出。当残差函数F优化为0时，网络实现了恒等映射，这从根本上解决了深层网络梯度消失的问题。

将残差连接引入Transformer后，研究者发现了两种主要变体：

变体 公式 优点 缺点
PostNorm h_l = Norm(h_{l-1} + F_l(h_{l-1})) 理论上表达能力更强 训练不稳定，层数增加时梯度急剧减小
PreNorm h_l = h_{l-1} + Norm(F_l(h_{l-1})) 训练更稳定，容易扩展到深层 存在"稀释效应"，隐藏状态幅度随深度增长

1.2 PreNorm的隐藏危机：稀释效应

2022年，DeepMind的论文"Understanding the Difficulty of Training Deep Transformers"深入分析了PreNorm的训练困难问题。研究发现：

核心问题：在PreNorm架构中，随着模型深度增加，所有层的输出以固定权重（=1）进行累加，这导致：
  • 隐藏状态幅度无限增长：每一层的输出直接加到累积和上，经过L层后，最终隐藏状态的范数约为 O(√L) 倍的初始幅度
  • 浅层特征被稀释：后期层看到的"信号"是前面所有层输出的混合，早期层学到的有意义的表示被后来的噪声稀释
  • 各层贡献呈指数衰减：虽然后期层对最终表示的直接贡献更大，但由于累加效应，浅层信息被深深埋藏在巨大的累积和中
  • 梯度分布不均：深层梯度范数远大于浅层，导致训练时各层学习速度差异巨大

PreNorm稀释效应的数学分析

经过L层PreNorm后，最终输出可展开为：

h_L = h_0 + Σ_{i=1}^{L} Norm(F_i(h_{i-1}))

关键观察：每一项的权重固定为1，这就是所谓的均匀聚合（Uniform Aggregation）。无论第i层的输出是否有用，都会被原封不动地累加到最终结果中。

这意味着：早期层学到的有意义的特征表示，会被后续层的输出"淹没"，因为它们被等权累加，无法区分哪些信息更重要。

二、核心创新：Attention Residuals

来自Kimi Team的论文Attention Residuals (AttnRes)提出了一个优雅而深刻的解决方案：用注意力机制替代固定的残差累加权重，让每个层能够输入依赖的、内容感知的方式选择性聚合前面的表示。

2.1 核心思想

AttnRes的核心突破在于：将"均匀聚合"替换为"选择性聚合"。具体而言：

AttnRes的核心公式

第l层的输出不再是简单相加，而是注意力加权求和：

h_l = Σ_{i=0}^{l-1} α_{i→l} · v_i

其中：

  • v_i = 第i层的输出表示（也包括词嵌入层的输出 v_0）
  • α_{i→l} = 从层i到层l的注意力权重（通过softmax归一化，确保 Σα = 1）

2.2 伪查询注意力机制详解

AttnRes的关键创新在于：如何计算注意力权重 α_{i→l}？论文提出了伪查询（Pseudo-Query）机制：

伪查询机制的实现

为每一层l引入一个可学习的伪查询向量 w_l ∈ ℝ^d：

# 伪查询注意力权重的计算
logit_{i→l} = (v_i · w_l) / √d   # 缩放点积注意力
α_{i→l} = softmax_i(logit_{i→l})  # 对所有前续层归一化

这与标准自注意力的区别在于：

  • 标准自注意力：Query来自当前输入，Key来自所有token
  • AttnRes：Query来自可学习的层特定参数，Key来自所有前续层的输出
设计动机：每一层的伪查询向量 w_l 学会了"询问"前面的层："对于我这一层的任务，哪些前续层的输出最有价值？"这种设计使得聚合过程真正实现了输入依赖——不同的层可以根据其任务需求选择性地关注不同来源的信息。

2.3 为什么这种方法有效？

AttnRes的有效性可以从多个角度理解：

（1）幅度控制

由于注意力权重经过softmax归一化，总和为1，因此：

  • 无论累加多少层，输出幅度始终保持有界
  • 解决了PreNorm中隐藏状态幅度随深度指数增长的问题

（2）选择性聚合

  • 模型可以学习哪些层的信息是"噪声"，降低其权重
  • 模型可以强化哪些层的信息是"信号"，提高其权重
  • 这种选择是输入依赖的——不同样本可能需要不同的聚合策略

（3）信息瓶颈缓解

  • 标准PreNorm中，所有层输出必须"挤在"同一个隐藏向量中
  • AttnRes允许模型选择性地保留/丢弃信息，实际上增加了有效信息容量

三、两种实现方案详解

3.1 Full AttnRes（完整版）

最直接的实现方式：每一层都可以 attends 到之前所有层的输出。

Full AttnRes的计算复杂度

对于L层模型，隐藏维度为d：

  • 每层需要存储所有前续层的输出：O(Ld) 额外内存
  • 每层的注意力计算：O(L²d) 计算量
  • 参数量：L × d（每个层一个伪查询向量）
问题：在超大模型（如L=80+层）上，O(Ld)的内存开销会变得不可接受。例如80层 × 8192维 = 655360额外存储，对于大规模训练这是显著瓶颈。

3.2 Block AttnRes（实用版）——核心工程创新

为解决内存问题，论文提出了分块版本的AttnRes，这是本文最重要的工程贡献：

分块策略

  • 将L层划分为N个块（block），每块包含 B = L/N 层
  • 块内：使用标准PreNorm残差连接（保持局部信息流动的效率）
  • 块间：使用注意力聚合（实现跨块选择性）

(a) 标准残差：所有层均匀累加 (b) Full AttnRes：每层 attends 到所有前续层 (c) Block AttnRes：块内标准残差，块间注意力聚合

Block AttnRes的内存优化

复杂度对比

方案 额外内存 计算复杂度
Standard PreNorm O(1) O(Ld)
Full AttnRes O(Ld) O(L²d)
Block AttnRes (N=8) O(Nd) O(N²d × B)

关键洞察：当N=8时，额外内存从O(Ld)降到O(Nd)，对于80层模型，内存开销减少10倍！

3.3 Block AttnRes的PyTorch实现

def block_attn_res(blocks: list[Tensor], partial_block: Tensor, 
                  proj: Linear, norm: RMSNorm) -> Tensor:
    """
    块间注意力机制的实现
    
    参数:
    - blocks: 已完成的N个块表示列表，每个[B, T, D]
    - partial_block: 当前块内的部分和 [B, T, D]
    - proj: 线性投影层 (用于生成伪查询)
    - norm: RMSNorm层 (用于Key归一化)
    
    返回:
    - 注意力加权聚合后的表示 [B, T, D]
    """
    # 堆叠所有块表示（包含当前部分和）
    V = torch.stack(blocks + [partial_block])  # [N+1, B, T, D]
    
    # 对Value进行归一化（类似QKNorm的思想）
    K = norm(V)
    
    # 计算伪查询（使用可学习投影）
    pseudo_query = proj.weight.squeeze()  # [d]
    
    # 计算注意力logits: (v_i · w_l) / √d
    logits = torch.einsum('d, n b t d -> n b t', pseudo_query, K)
    # logits shape: [N+1, B, T]
    
    # Softmax归一化（沿块维度）
    attention_weights = logits.softmax(0)  # [N+1, B, T]
    
    # 加权求和
    h = torch.einsum('n b t, n b t d -> b t d', attention_weights, V)
    return h

3.4 关键实现细节

设计决策 选择 理由
每层两个AttnRes 在attention和MLP前各应用一次 最大化信息选择性，每个子层都可以利用历史信息
使用RMSNorm 对Value进行归一化 保持数值稳定性，类似QKNorm的思想
伪查询结构 使用单层线性投影 [1, d] → [d] 最小化参数量，同时保留足够的表达能力
块大小 推荐约8个块（N=8） 实验发现8个块能恢复大部分Full AttnRes收益

AttnRes (橙色) vs Baseline (蓝色) 在不同计算预算下的 loss 对比

关键发现

  • 全面超越：AttnRes在所有计算预算下都一致优于基线
  • 1.25x计算等效：Block AttnRes的性能相当于用1.25倍计算资源训练的基线模型
  • 收益稳定性：随着计算预算增加，收益保持稳定，没有衰减迹象

收益的量化计算

假设基线在计算预算C下的性能为L_baseline(C)，AttnRes在相同预算下的性能为L_attn(C)。实验发现：

L_attn(C) ≈ L_baseline(1.25 × C)

这意味着：使用相同的计算资源，AttnRes可以获得约25%的性能提升。从另一个角度看，达到相同的性能，AttnRes可以节省20%的训练计算。

4.3 结果深度分析

（1）最大收益来源：多步推理

关键发现：GPQA-Diamond（研究生水平问答）获得了最大的绝对收益（+7.5，+20.3%），这远超其他任务。

分析原因：

  • GPQA-Diamond需要多步推理和深度理解，涉及复杂的逻辑链
  • AttnRes的选择性聚合使模型能够更好地保留和利用早期层学到的中间推理步骤
  • 标准PreNorm中，这些中间信息会被后续层的输出"稀释"，难以被有效利用
  • AttnRes允许模型学习在每一步推理中关注相关的历史信息

（2）代码生成显著提升

  • HumanEval 提升+3.1（+5.2%）：代码生成需要精确执行每一步操作
  • 这与多步推理的逻辑类似：代码也是一系列需要精确组合的步骤
  • AttnRes帮助模型更好地跟踪和组合代码的不同部分

（3）数学推理

  • Math 提升+3.6（+6.7%）：数学推理也是典型的多步推理任务
  • 需要保留和组合多个中间计算结果
  • AttnRes的选择性聚合正好符合这种需求

（4）基础语言理解任务

  • MMLU、TriviaQA 也有稳定提升（+1.1~+1.9）
  • 这些任务对信息选择的要求相对较低，但仍有收益
  • 说明AttnRes的幅度控制效应对所有任务都有普遍帮助

4.4 训练动态分析

论文深入分析了AttnRes对训练过程的影响：

左图：输出幅度随层数变化；右图：梯度范数随层数变化

（1）输出幅度控制

标准PreNorm的问题：

  • 输出幅度随层数指数增长
  • 40层后，输出范数可能是初始值的10倍以上
  • 导致需要非常小的学习率，否则训练不稳定

AttnRes的解决：

  • 注意力权重的softmax归一化自然约束了输出幅度
  • 无论聚合多少层的信息，输出范数保持在合理范围内
  • 允许使用更大的学习率，加快训练收敛

（2）梯度分布优化

标准PreNorm的问题：

  • 深层梯度范数远大于浅层
  • 浅层参数更新不足，学习缓慢
  • 深层容易过拟合，浅层表示不够充分

AttnRes的改进：

  • 梯度在各层之间分布更均匀
  • 每层都能得到充分的训练信号
  • 整个网络学习更均衡

六、总结与展望

6.1 主要贡献

  1. 理论贡献：首次深入分析了PreNorm中"均匀聚合"的问题，指出这导致了信息稀释和幅度增长
  2. 方法创新：提出用注意力机制替代固定权重，实现了真正的"选择性聚合"
  3. 工程优化：Block AttnRes在几乎无额外开销的情况下实现了~25%计算等效收益
  4. 实验验证：在真实的大规模模型（48B参数）上验证了方法的有效性

6.2 核心亮点

  • 简单而深刻：只改变了一个基础组件，但带来了系统性的性能提升
  • 即插即用：不需要改变模型架构，可以轻松集成到现有预训练流程
  • 理论解释：从幅度控制、选择性聚合、信息瓶颈等多个角度解释了其有效性
  • 实用性：Block AttnRes的内存开销极小，适合大规模训练

6.3 启发

这篇论文提醒我们：在深度学习领域，一些看似"理所当然"的设计（如固定权重残差）可能隐藏着可以优化的空间。AttnRes的成功源于它对基础组件的重新思考。
