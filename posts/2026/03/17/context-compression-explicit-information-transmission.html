一、引言：长上下文推理的困境

1.1 为什么长上下文如此昂贵？

随着大语言模型（LLM）的规模不断扩大，处理长上下文输入的计算和内存成本已成为实际部署的主要瓶颈。造成这种高成本的原因主要有两点：

  • 二次方复杂度的自注意力：标准Transformer的自注意力机制需要计算所有token对之间的注意力分数，对于长度为N的序列，计算复杂度为O(N²)
  • 不断增长的KV缓存：在自回归生成过程中，需要存储所有已处理token的Key和Value向量，内存占用随序列长度线性增长
现实问题：处理一篇100K token的文档可能需要数十GB的GPU显存，这在实际应用中几乎不可行。

1.2 上下文压缩的兴起

为了解决上述问题，上下文压缩（Context Compression）技术应运而生。其核心思想是将长上下文压缩成少量紧凑的向量表示，从而降低推理成本。

现有的压缩方法可以分为两大类：

类别 核心思想 优点 缺点
硬压缩 根据重要性评分直接删除token 实现简单，计算开销小 离散、有损，高压缩率下信息丢失严重
软压缩 将多个token的信息聚合到连续向量中 表达能力强，聚合粒度细 现有方法存在结构性问题

1.3 现有软压缩方法的问题

大多数现有的软压缩方法采用LLM-as-a-compressor范式：通过引入特殊的压缩token（gist/memory tokens），利用LLM内部的自注意力机制逐层迭代地从上下文token中吸收信息。

论文核心发现：这种范式存在两个关键的结构性缺陷！

问题一：逐层表示覆盖（Representation Overwriting）

在逐层迭代更新过程中，压缩token在早期层聚合的信息可能被后续层的更新所覆盖。同时，深层表示会逐渐偏向高度抽象的、专门用于生成的特征，导致：

  • 压缩器与解码器之间存在分布失配（Distribution Mismatch）
  • 压缩优化更加困难，性能下降

问题二：缺乏全局协调（Lack of Global Coordination）

自注意力的聚合是各压缩token独立进行的，没有机制来协调不同token之间的容量分配：

  • 多个压缩token可能重复关注同一批上下文位置
  • 其他重要位置被忽视
  • 有限的压缩预算被低效利用

二、ComprExIT：显式信息传输框架

来自Jiangnan Ye等人发表的论文"Context Compression via Explicit Information Transmission"提出了ComprExIT（Context Compression via Explicit Information Transmission），这是一个全新的压缩范式。

核心思想

ComprExIT将压缩过程从LLM内部的自注意力机制中解耦出来，转而对冻结LLM的隐藏状态进行显式信息传输。这种方法：

  • 避免逐层表示漂移
  • 实现全局协调的信息分配
  • 只引入约1%的额外参数

2.1 两种传输维度

ComprExIT沿两个正交维度执行信息传输：

（1）深度传输（Depth-wise Transmission）

沿层维度，选择性地将多层隐藏状态信息传输到token锚点，缓解逐层覆盖问题。

（2）宽度传输（Width-wise Transmission）

沿token维度，将N个token锚点聚合成K个压缩槽位，通过全局优化的传输计划确保信息协调分配。

2.2 Token锚点的构建（深度传输）

首先，需要将LLM各层的隐藏状态聚合为token锚点。ComprExIT采用以下步骤：

步骤1：结构化层混合

对每个token位置t，首先进行层状态的加权混合：

层混合公式

h̄_t = Σ_{ℓ=1}^{L} w_ℓ · h_t^{(ℓ)}, Σ_{ℓ=1}^{L} w_ℓ = 1

其中 w_ℓ 是可学习的层优先级权重。

步骤2：层门控计算

基于混合后的上下文，计算层门控分数：

门控分数计算

s_{t,ℓ} = ⟨W_c · h̄_t, W_ℓ · h_t^{(ℓ)} + e_ℓ⟩

其中 e_ℓ 是可学习的层嵌入。

步骤3：门控归一化与聚合

对门控分数进行softmax归一化（温度τ控制）：

门控系数

α_{t,ℓ} = exp(s_{t,ℓ}/τ) / Σ_{j=1}^{L} exp(s_{t,j}/τ)

最终生成token锚点：

h̃_t = Σ_{ℓ=1}^{L} α_{t,ℓ} · W_a · h_t^{(ℓ)}

设计动机：不同token可以根据其语义重要性选择不同层的表示。重要实体token倾向于选择中间层（包含更丰富的上下文和关系信息），而不重要的token则更依赖早期层（捕获局部词汇特征）。

2.3 压缩槽位的构建（宽度传输）

将N个token锚点压缩成K个槽位（K << N）。这是通过最优传输（Optimal Transport）理论来实现的。

步骤1-4：构建接收者、效用矩阵、信息容量、求解最优传输计划

通过优化以下问题得到全局协调的传输计划：

最优传输问题

最小化：Σ_{t,k} Π_{t,k} · C_{t,k}

约束条件：

  • Σ_k Π_{t,k} = ρ_t, ∀t（发送者容量约束）
  • Σ_t Π_{t,k} = ρ_k, ∀k（接收者容量约束）

论文使用Sinkhorn算法高效求解这个问题。

2.4 最终压缩表示

每个压缩槽位根据传输计划聚合信息，然后通过一个轻量级MLP进行表示对齐，生成最终的压缩表示。

2.5 两阶段训练方法

ComprExIT采用两阶段训练：

阶段 目标 数据
第一阶段：NTP 下一个token预测，保持通用上下文信息 SlimPajama (1B tokens)
第二阶段：SFT 下游任务微调，适应任务相关的选择性压缩 MRQA (6个in-domain + 6个out-of-domain QA数据集)

三、实验结果与分析

3.1 实验设置

模型与配置

  • 基础模型：Llama-3.2-1B-Base 和 Llama-3.2-3B-Base
  • 投影隐藏维度：256
  • Sinkhorn迭代次数：30
  • 压缩率：4倍（512 → 128 tokens）
  • 评估指标：Exact Match (EM) 和 F1

3.3 关键发现

  • ComprExIT在所有6个QA基准上一致优于所有压缩基线方法
  • 在1B模型上平均EM提升8.18%（52.34 vs 44.16）
  • 在多个数据集上（TriviaQA、SearchQA等）， ComprExIT超越了无压缩的prompt tuning基线
  • 当从3B缩小到1B时， ComprExIT保持稳定，而500×和Beacon方法性能急剧下降

3.5 进一步分析

（1）协调分配的效果

  • ICAE：压缩token之间存在显著的重叠（高Pearson相关系数），表明多个token重复关注同一内容

四、总结与思考

4.1 主要贡献

  1. 识别问题：首次系统性地指出LLM-as-a-compressor范式的两个关键结构性缺陷
  2. 方法创新：提出 ComprExIT，将软压缩重新定义为冻结LLM隐藏状态上的显式信息传输问题
  3. 工程优化：引入深度传输和宽度传输两个维度，通过最优传输实现全局协调
  4. 显著效果：在6个QA基准上始终优于现有方法，额外参数仅约1%

4.2 核心亮点

  • 解耦设计：将压缩与LLM内部自注意力解耦，避免逐层表示漂移
  • 全局优化：通过最优传输理论实现信息分配的全局协调
  • 高效轻量：只引入约1%的额外参数
  • 稳定可扩展：在小模型上性能稳定
