一、引言：长上下文推理的困境

1.1 为什么长上下文如此昂贵？

随着大语言模型（LLM）的规模不断扩大，处理长上下文输入的计算和内存成本已成为实际部署的主要瓶颈。造成这种高成本的原因主要有两点：

  • 二次方复杂度的自注意力：标准Transformer的自注意力机制需要计算所有token对之间的注意力分数
  • 不断增长的KV缓存：在自回归生成过程中，需要存储所有已处理token的Key和Value向量

1.2 上下文压缩的兴起

现有的压缩方法可以分为两大类：

类别 核心思想 优点 缺点
硬压缩 根据重要性评分直接删除token 实现简单，计算开销小 离散、有损
软压缩 将多个token的信息聚合到连续向量中 表达能力强 现有方法存在结构性问题

1.3 现有软压缩方法的问题

大多数现有的软压缩方法采用LLM-as-a-compressor范式：通过引入特殊的压缩token，利用LLM内部的自注意力机制逐层迭代地从上下文token中吸收信息。

论文核心发现：这种范式存在两个关键的结构性缺陷！

图1：现有方法的两个结构性弱点
（左）表示覆盖问题：捕获2012年份信息的压缩token在逐层编码过程中信息逐渐被覆盖
（右）缺乏全局协调：关键token"Paris"可能完全不被任何压缩token关注

问题一：逐层表示覆盖

在逐层迭代更新过程中，压缩token在早期层聚合的信息可能被后续层的更新所覆盖，导致压缩器与解码器之间存在分布失配。

问题二：缺乏全局协调

自注意力的聚合是各压缩token独立进行的，没有机制来协调不同token之间的容量分配，导致信息冗余和遗漏。

二、ComprExIT：显式信息传输框架

来自Jiangnan Ye等人发表的论文提出了ComprExIT（Context Compression via Explicit Information Transmission），这是一个全新的压缩范式。

核心思想

ComprExIT将压缩过程从LLM内部的自注意力机制中解耦出来，转而对冻结LLM的隐藏状态进行显式信息传输。这种方法：

  • 避免逐层表示漂移
  • 实现全局协调的信息分配
  • 只引入约1%的额外参数

图2：现有方法 vs ComprExIT 架构对比
左：现有LLM-as-a-compressor方法通过gist tokens迭代编码
右：ComprExIT利用冻结LLM的隐藏状态，通过协调的传输计划进行压缩

2.1 两种传输维度

ComprExIT沿两个正交维度执行信息传输：

（1）深度传输（Depth-wise Transmission）

沿层维度，选择性地将多层隐藏状态信息传输到token锚点，缓解逐层覆盖问题。

（2）宽度传输（Width-wise Transmission）

沿token维度，将N个token锚点聚合成K个压缩槽位，通过全局优化的传输计划确保信息协调分配。

2.2 Token锚点的构建（深度传输）

首先，需要将LLM各层的隐藏状态聚合为token锚点：

层混合公式

h̄_t = Σ_{ℓ=1}^{L} w_ℓ · h_t^{(ℓ)}

然后通过门控机制选择性地聚合各层信息：

h̃_t = Σ_{ℓ=1}^{L} α_{t,ℓ} · W_a · h_t^{(ℓ)}

设计动机：不同token可以根据其语义重要性选择不同层的表示。重要实体token倾向于选择中间层（包含更丰富的上下文和关系信息）。

2.3 压缩槽位的构建（宽度传输）

将N个token锚点压缩成K个槽位（K << N）。通过最优传输（Optimal Transport）理论来求解全局协调的传输计划。

最优传输问题

最小化：Σ_{t,k} Π_{t,k} · C_{t,k}

约束：发送者和接收者的容量守恒

使用Sinkhorn算法高效求解

三、实验结果与分析

3.1 实验设置

模型与配置

  • 基础模型：Llama-3.2-1B-Base 和 Llama-3.2-3B-Base
  • 压缩率：4倍（512 → 128 tokens）
  • 评估指标：Exact Match (EM) 和 F1

3.2 主实验结果

Prompt tuning 71.89 30.72 61.04 64.03 53.60 53.03 55.72

3.3 关键发现

  • ComprExIT在所有6个QA基准上一致优于所有压缩基线方法
  • 平均EM提升8.18%（52.34 vs 44.16）
  • 在多个数据集上超越了无压缩的prompt tuning基线
  • 在小模型（1B）上性能稳定，而其他方法急剧下降

四、总结与思考

4.1 主要贡献

  1. 识别问题：首次系统性地指出LLM-as-a-compressor范式的两个关键结构性缺陷
  2. 方法创新：提出 ComprExIT，将软压缩重新定义为冻结LLM隐藏状态上的显式信息传输
  3. 工程优化：引入深度传输和宽度传输，通过最优传输实现全局协调
  4. 显著效果：在6个QA基准上始终优于现有方法，额外参数仅约1%

4.2 核心亮点

  • 解耦设计：将压缩与LLM内部自注意力解耦
  • 全局优化：通过最优传输理论实现信息分配的全局协调
  • 高效轻量：只引入约1%的额外参数
  • 稳定可扩展：在小模型上性能稳定
